# UE5 Graph Framework & Editor Tools

Тренировочный проект на Unreal Engine 5, направленный на разработку графовой системы (Graph Framework) и кастомных инструментов редактора.

## Обзор

Проект реализует модульную графовую архитектуру, используемую для:

* квестовой логики
* диалоговых систем
* системы скиллов

Включает как runtime-часть, так и инструменты редактора (Editor Tools), приближенные к реальному production-пайплайну Unreal Engine.

---

## Цели проекта

* Спроектировать расширяемую graph-based архитектуру
* Реализовать кастомные инструменты для Unreal Editor
* Построить node-based workflow для дизайнеров
* Отработать работу с C++ и внутренностями Unreal Engine
* Практика оптимизации и профилирования

---

## Основной функционал

### Graph Framework (Runtime)

* Кастомный graph asset (`UGFGraphAsset`)
* Архитектура: ноды, пины, связи
* Система выполнения графа (execution)
* Расширяемость под разные домены (quest / dialogue / skill)

---

### Editor Tools

* Кастомный тип ассета
* Собственный редактор графа
* Node-based интерфейс (GraphEditor API)
* Контекстное меню добавления нод
* Кастомизация Details Panel
* Валидация графа и инструменты отладки

---

### Системы поверх графа

#### Квесты

* условия
* действия
* ветвление

#### Диалоги

* реплики
* выборы
* условные переходы

#### Скиллы

* последовательность выполнения
* проверки (cooldown, условия)
* эффекты

---

### Внешние параметры

* Интеграция с внешними системами (например, погода)
* Ноды, взаимодействующие с внешним состоянием

---

### Производительность и отладка

* Профилирование выполнения графа
* Валидация структуры графа
* Отладка выполнения (execution trace)

---

### Тестирование

* Базовые automated tests
* Проверка сериализации
* Проверка корректности графа

## Рабочий процесс

Типичный цикл работы:

1. Открыть graph asset в Unreal Editor
2. Изменить структуру графа (ноды, связи)
3. При необходимости изменить C++ код
4. Пересобрать проект
5. Проверить выполнение и валидацию
6. Повторить цикл
